# /cleanup-artifacts — Сведение, разбивка и чистка

## Использование

```
/cleanup-artifacts <game> <version>
```

Пример: `/cleanup-artifacts brp srd-1.0`

## Предварительные условия

Файлы из `/convert-pdf` должны существовать в `/tmp/`:
- `/tmp/{game}_marker.md`
- `/tmp/{game}_pymupdf.md`
- `/tmp/{game}_docling.md`

Также нужен оригинальный PDF для сверки (спросить путь если неизвестен).

## Алгоритм

### Фаза A: Сведение (merge)

Чеклист: **`.claude/rules/merge-extraction.md`** — правила выбора лучших частей из конвертеров.

Создай **Agent** (обязательно **model: "opus"**) для сведения. Агент ОБЯЗАН прочитать rules файл перед началом работы.

Агент анализирует все три результата конвертации + оригинальный PDF по правилам из rules файла.

3. **Собери единый markdown** — `/tmp/{game}_merged.md`

4. **Отчёт о полезности конвертеров:**

После сведения выведи статистику:

```
Полезность конвертеров для {game} {version}:

marker:
  - Использовано для: N% текста, N% таблиц, N% заголовков
  - Сильные стороны: ...
  - Слабые стороны: ...

pymupdf4llm:
  - Использовано для: N% текста, N% таблиц, N% заголовков
  - Сильные стороны: ...
  - Слабые стороны: ...

docling:
  - Использовано для: N% текста, N% таблиц, N% заголовков
  - Сильные стороны: ...
  - Слабые стороны: ...

Итого: {основной источник} — основной, {второй} — для таблиц, {третий} — верификация
```

Проценты рассчитывай по количеству строк, взятых из каждого конвертера.

### Фаза B: Layout Recovery (восстановление структуры)

Чеклист: **`.claude/rules/layout-recovery.md`**

Агент восстанавливает структуру документа: склейка разорванных абзацев и слов, удаление номеров страниц и колонтитулов, OCR-маркеров, image placeholders, нормализация заголовков, таблиц, списков.

Выполняется на `/tmp/{game}_merged.md` перед разбивкой на файлы.

### Фаза C: Разбивка на файлы

1. **Определи точки разбивки** по структуре документа:
   - Каждая глава/крупный раздел → отдельный файл
   - `00_Legal.md` — лицензия/OGL всегда первым файлом
   - Нумерация: `00_Legal.md`, `01_Name.md`, `02_Name.md`, ...

2. **Именование по паттерну проекта** — см. `.claude/rules/file-naming-conventions.md`

3. **Глоссарий НЕ создаётся** (это задача `/build-glossary`)

4. **Коммит после каждого файла:**
   ```
   Импорт {game} {version}: {filename}
   ```

### Фаза D: Markdown Normalization (нормализация)

Чеклист: **`.claude/rules/pdf-cleanup.md`**

Механическая чистка форматирования: лигатуры, soft hyphens, пробелы, тире, кавычки, пустые строки. Выполняется для каждого созданного файла.

**Коммит после нормализации каждого файла:**
```
Импорт {game} {version}: нормализация {filename}
```

### Финальная сводка

```
Сведение и чистка завершены:

Файлы созданы: K
- src/{game}/{version}/en/00_Legal.md
- src/{game}/{version}/en/01_....md
- ...

Артефактов исправлено: M
- Разрывов строк: ...
- Номеров страниц: ...
- Сломанных таблиц: ...
- Проблем форматирования: ...

Следующий шаг: /verify-import {game} {version}
```

## Технические требования

- Агент сведения — **model: "opus"** (обязательно)
- Большие файлы (>3000 строк) — чтение чанками через offset/limit
- Коммит после каждого файла (не групповой)
- Сообщения коммитов на русском
- Оригинальный PDF нужен для сверки при расхождениях
