# /translate-verify — Верификация глоссария

## Использование

```
/translate-verify <game> <version>
```

Пример: `/translate-verify daggerheart srd-1.0`

## Алгоритм

### Шаг 1: Подготовка

1. Найди EN глоссарий: `src/{game}/{version}/en/*_Glossary/`
2. Найди RU глоссарий: `src/{game}/{version}/ru/*_Glossary/`
3. Если RU не найден — предложи сначала `/translate-glossary`

## Правила проверки

Чеклисты:
- **`.claude/rules/translation-validation.md`** — структура, терминология, оформление, локализация
- **`.claude/rules/translation-quality-review.md`** — редакторская проверка качества (для `verifier-quality`)

Агенты ОБЯЗАНЫ прочитать свои rules файлы перед началом проверки. Для глоссария применяются разделы **1-5** (общие) + раздел **6** (специфика глоссария) + раздел **7** (кросс-версионная консистентность).

### Шаг 2: Создание команды верификаторов

Используй **TeamCreate** для создания команды `verify-glossary-{game}`.

Создай 3 агента через **Agent** (все с **model: "opus"**):

#### Агент 1: `verifier-structure`

**Фокус:** Структурная целостность

Проверяет разделы **1. Структурное соответствие** + **6. Специфика глоссария → Структура, Точность** из `.claude/rules/translation-validation.md`.

#### Агент 2: `verifier-consistency`

**Фокус:** Консистентность терминологии

Проверяет разделы **2. Терминологическое соответствие** + **6. Специфика глоссария → Консистентность** + **7. Кросс-версионная консистентность** из `.claude/rules/translation-validation.md`.

#### Агент 3: `verifier-quality`

**Фокус:** Качество, оформление и локализация

Проверяет разделы **3. Качество оформления** + **5. Качество локализации** + **6. Специфика глоссария → Оформление** из `.claude/rules/translation-validation.md`.

Дополнительно использует **`.claude/rules/translation-quality-review.md`** как системный промпт — жёсткая редакторская проверка: кальки, канцелярит, тяжеловесность, смысловые смещения.

### Шаг 3: Цикл верификации (минимум 3 раунда, до 0 ошибок)

```
раунд = 1
ПОВТОРЯТЬ:
    1. Запустить всех 3 агентов ПАРАЛЛЕЛЬНО (через TaskCreate + Agent)
    2. Собрать результаты от всех агентов
    3. Объединить в общий отчёт:
       - Ошибки структуры: [список]
       - Ошибки консистентности: [список]
       - Ошибки качества: [список]
    4. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - Коммит: "Глоссарий {game} {version}: исправления (раунд {N})"
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл (даже если раунд > 3 — продолжать пока есть ошибки)
    5. ЕСЛИ ошибок нет И раунд >= 3:
       - ВЫХОД из цикла ✓
    6. ЕСЛИ ошибок нет И раунд < 3:
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл (нужно минимум 3 раунда)
```

**Условие выхода:** раунд >= 3 И ошибок = 0. Цикл НЕ завершается при наличии ошибок, даже если раундов уже больше 3.

### Шаг 4: Отчёт

```
Верификация глоссария завершена:
- Раундов: N
- Исправлений: M
- Статус: ✓ Все проверки пройдены

Детали по раундам:
- Раунд 1: 5 ошибок структуры, 3 консистентности, 2 качества → исправлено
- Раунд 2: 1 ошибка консистентности → исправлено
- Раунд 3: 0 ошибок ✓
```

### Шаг 5: Пауза для ручной проверки

После успешной автоверификации:

1. Используй **AskUserQuestion**:
   - Вопрос: "Автоверификация глоссария завершена (N раундов, M исправлений). Проверьте файлы в `src/{game}/{version}/ru/*_Glossary/` и скажите ОК или внесите правки."
   - Опции: "ОК, продолжаем" / "Внёс правки, перепроверить"

2. Если пользователь выбрал "Внёс правки":
   - Запустить полный цикл верификации заново (минимум 3 раунда)
   - Снова показать AskUserQuestion
   - Повторять пока не скажет "ОК"

3. Если пользователь выбрал "ОК" — завершить

## Терминологические решения

Если в ходе верификации обнаружен спорный перевод термина:

1. Прочитай `src/{game}/translate/decisions.md` и `src/translate/decisions.md` (общие) — возможно, решение уже задокументировано
2. Если решение уже есть и агент предлагает другой вариант — решение из лога имеет приоритет
3. Если решение новое (термин не в логе) — предложить пользователю и при подтверждении **добавить запись** в `src/{game}/translate/decisions.md`

## Технические требования

- Все 3 агента — **model: "opus"** (обязательно)
- Агенты запускаются **параллельно** через TeamCreate
- Минимум 3 раунда проверки даже если ошибок нет
- Коммит только когда есть реальные исправления
- Сообщения коммитов на русском
- После автоверификации — обязательная пауза (AskUserQuestion)
