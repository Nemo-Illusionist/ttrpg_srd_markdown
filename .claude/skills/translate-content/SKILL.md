# /translate-content — Перевод контента по глоссарию

## Использование

```
/translate-content <game> <version>
```

Пример: `/translate-content daggerheart srd-1.0`

## Алгоритм

### Шаг 1: Построение словаря терминов

1. Найди RU глоссарий: `src/{game}/{version}/ru/*_Glossary/`
2. Прочитай ВСЕ файлы глоссария
3. Построй маппинг `{EN: RU}` из всех таблиц:
   - Колонка "Оригинал" → EN ключ
   - Первая колонка → RU значение
4. Выведи статистику: `Словарь: N терминов из M файлов`

### Шаг 2: Определение файлов для перевода

1. Найди все EN файлы: `src/{game}/{version}/en/**/*.md`
2. Исключи файлы глоссария (`*_Glossary/`)
3. Определи порядок перевода:
   - `00_Legal*` → первым (лицензия)
   - Затем по номерам (`01_`, `02_`, ...)
   - Подпапки: файлы внутри по номерам

```
Файлы для перевода: K файлов
- 00_Legal.md
- 01_Characters.md
- 02_Equipment.md
- 03_Classes/01_Barbarian.md
- 03_Classes/02_Bard.md
...
```

### Шаг 3: Перевод каждого файла

Для каждого файла (в определённом порядке):

1. **Прочитай EN файл** (большие — чанками с offset/limit, максимум 3000 строк)
2. **Переведи** с обязательным использованием словаря терминов:
   - Каждый EN термин из словаря → заменяется на RU эквивалент
   - Неизвестные термины → перевод по контексту, но выведи предупреждение
3. **Сохрани форматирование:**
   - Уровни заголовков (`#`, `##`, `###`, `####`) — количество и иерархия = EN
   - Bold/italic (`**text**`, `*text*`, `**_text_**`) — сохранить разметку
   - Blockquotes (`>`) — сохранить структуру (stat blocks)
   - Таблицы — количество строк и столбцов = EN
   - Списки — количество элементов = EN
   - Ссылки `[text](url)` — перевести text, url без изменений
   - Пустые строки — сохранить структуру
4. **Создай RU файл** в зеркальной структуре: `src/{game}/{version}/ru/{same_path}`
5. **Верификация** — запусти opus-агент через **Agent** (model: "opus"):
   - Агент читает `.claude/rules/translation-validation.md` и проверяет пару EN/RU файлов по ВСЕМУ чеклисту
   - Передай агенту: пути к EN и RU файлам, словарь терминов из глоссария
6. **Fix-цикл:**
   - Если агент нашёл ошибки → исправить → перезапустить верификацию
   - Повторять до 0 ошибок
7. **Коммит** (только после чистой верификации): `Перевод {game} {version}: {filename}`

### Шаг 4: Итоговая проверка

1. Подсчитай файлы:
   - EN файлов (без глоссария): X
   - RU файлов (без глоссария): Y
   - X = Y — ОК
2. Выведи итог:

```
Перевод завершён:
- Файлов переведено: K
- Предупреждения о неизвестных терминах: N (если есть)
- Структура: все проверки пройдены ✓
```

## Правила перевода

### Обязательные

- **Словарь — закон.** Каждый термин из глоссария переводится ТОЛЬКО по словарю
- **Структура = EN.** Количество заголовков, строк таблиц, элементов списков идентично
- **Форматирование неприкосновенно.** Bold, italic, blockquotes, ссылки — без потерь

### Стиль

- Игромеханические термины с заглавной буквы (как в глоссарии)
- Числа без изменений
- Единицы измерения: feet → футов, pounds → фунтов (если не в глоссарии — по стандарту D&D/TTRPG)
- Имена собственные → транслитерация (если не в глоссарии)

### Большие файлы (>3000 строк)

1. Читай чанками: offset=0 limit=3000, offset=3000 limit=3000, ...
2. Переводи чанками, сохраняя контекст между ними
3. Собирай полный RU файл перед записью

## Технические требования

- Все агенты используют **model: "opus"**
- Большие файлы — чанками через offset/limit
- Коммит после КАЖДОГО файла (не групповой)
- Зеркальная структура папок RU = EN
- Сообщения коммитов на русском
