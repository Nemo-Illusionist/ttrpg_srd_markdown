# /validate-markdown — Валидация структуры Markdown

## Использование

```
/validate-markdown <path>
/validate-markdown <path_EN> <path_RU>
```

- `/validate-markdown src/brp/srd-1.0/en/` — проверка одного набора файлов
- `/validate-markdown src/brp/srd-1.0/en/ src/brp/srd-1.0/ru/` — парная проверка EN↔RU

## Назначение

Универсальный структурный валидатор markdown. Используется:
- После **импорта** (`/cleanup-artifacts`) — проверка чистоты EN файлов
- После **перевода** (`/translate-content`) — проверка сохранности структуры EN↔RU
- Самостоятельно — проверка любых markdown файлов

## Алгоритм

### Шаг 1: Сбор файлов

1. Найди все `.md` файлы по указанному пути (рекурсивно)
2. Если указаны два пути — построй пары `{EN_file} ↔ {RU_file}` по одинаковым относительным путям
3. Файлы без пары — отдельная ошибка

### Шаг 2: Проверка каждого файла

Для каждого файла проверь:

#### 2.1. Таблицы

- Все строки таблицы имеют одинаковое количество `|`
- Разделительная строка (`|---|---|`) существует после заголовка
- Разделительная строка содержит то же количество колонок, что и заголовок
- Нет разорванных таблиц (строка таблицы, потом пустая строка, потом снова строка таблицы)
- Нет ячеек с незакрытым форматированием внутри таблицы

#### 2.2. Заголовки

- Иерархия не нарушена (нет `####` сразу после `#`, без промежуточных `##` и `###`)
- Перед заголовком есть пустая строка (кроме первой строки файла)
- После `#` стоит пробел (`#Заголовок` → ошибка)
- Нет дублирующихся заголовков одного уровня подряд

#### 2.3. Списки

- Вложенные списки имеют корректный отступ (2 или 4 пробела, единообразно в файле)
- Нет смешивания маркеров (`-` и `*`) в одном списке
- Перед списком есть пустая строка

#### 2.4. Форматирование

- Все `**` парные (нет незакрытого bold)
- Все `*` парные (нет незакрытого italic), с учётом `**` и `***`
- Все `` ` `` парные (нет незакрытого inline code)
- Все `[` имеют парную `]` и `(url)` (нет битых ссылок вида `[text]` без `(url)`)
- Нет `**_text_**` с пробелом внутри маркеров (`** text **`)
- Blockquotes (`>`) — нет строки без `>` внутри blockquote-блока

#### 2.5. Пробелы и спецсимволы

- Нет двойных пробелов (кроме конца строки для line break)
- Нет trailing whitespace
- Нет строк только из пробелов
- Нет 3+ пустых строк подряд
- Нет табуляций (если проект использует пробелы)
- Тире: ` — ` (длинное с пробелами), а не ` -- ` или ` - ` внутри предложений

### Шаг 3: Парная проверка EN↔RU (если два пути)

Для каждой пары файлов:

- Количество заголовков каждого уровня: RU = EN
- Количество строк таблиц: RU = EN
- Количество столбцов таблиц: RU = EN (или EN+1 для глоссария с колонкой "Оригинал")
- Количество элементов списков: RU = EN
- Количество blockquotes: RU = EN
- Количество строк: RU ≈ EN (допуск ±5%)

### Шаг 4: Отчёт

```
Валидация markdown завершена:

Файлов проверено: K
Ошибок найдено: N

По категориям:
- Таблицы: X ошибок
- Заголовки: Y ошибок
- Списки: Z ошибок
- Форматирование: W ошибок
- Пробелы: V ошибок
- Парная проверка: U ошибок

Детали:
- src/.../01_File.md:42 — таблица: строка 42 имеет 4 колонки, заголовок — 3
- src/.../02_File.md:15 — заголовок: #### после #, пропущены ## и ###
- ...

Статус: ✓ Все проверки пройдены / ✗ Найдены ошибки
```

## Автоисправление

Если вызван с флагом `--fix`:

```
/validate-markdown src/brp/srd-1.0/en/ --fix
```

После обнаружения ошибок автоматически исправляет:
- Двойные пробелы → одинарные
- Trailing whitespace → удалить
- 3+ пустых строк → 2
- ` -- ` → ` — `
- `#Заголовок` → `# Заголовок`
- Пустая строка перед заголовком (если отсутствует)

**Не исправляет автоматически** (только сообщает):
- Битые таблицы (нужен контекст)
- Незакрытое форматирование (неоднозначность)
- Нарушение иерархии заголовков (нужно решение)
- Парные расхождения EN↔RU (нужен перевод)

Коммит после автоисправления:
```
Валидация markdown: автоисправление {path}
```

## Технические требования

- Большие файлы (>3000 строк) — чтение чанками через offset/limit
- Парная проверка — агент читает оба файла параллельно
- Сообщения коммитов на русском
