# /verify-content — Верификация перевода контента

## Использование

```
/verify-content <game> <version>
```

Пример: `/verify-content daggerheart srd-1.0`

## Алгоритм

### Шаг 1: Подготовка

1. Найди EN контент: `src/{game}/{version}/en/**/*.md` (исключая `*_Glossary/`)
2. Найди RU контент: `src/{game}/{version}/ru/**/*.md` (исключая `*_Glossary/`)
3. Построй словарь терминов из RU глоссария (как в `/translate-content`)
4. Если RU контент не найден — предложи сначала `/translate-content`

### Шаг 2: Создание команды верификаторов

Используй **TeamCreate** для создания команды `verify-content-{game}`.

Создай 3 агента через **Agent** (все с **model: "opus"**):

#### Агент 1: `verifier-terminology`

**Фокус:** Терминологическая точность и количественное соответствие

Проверяет:

**A. Правильность терминов:**
- Извлечь все пары EN→RU из глоссария (`*_Glossary/00_Glossary.md`)
- Для каждого термина: в RU файле используется ТОЛЬКО утверждённый RU перевод
- **Grep** EN терминов в RU файлах — не должно быть совпадений (кроме: proper nouns, URL, code blocks)
- Один и тот же термин переведён одинаково во ВСЕХ файлах
- Нет синонимов (один термин переведён двумя способами в разных файлах)
- Термины из `00_Glossary.md` — авторитетный источник

**B. Количественное соответствие EN↔RU (КРИТИЧНО):**
- Для каждого термина из глоссария посчитать:
  - Кол-во вхождений EN термина в каждом EN файле
  - Кол-во вхождений RU термина в соответствующем RU файле
- **Числа ДОЛЖНЫ совпадать** для каждой пары файлов
- Если не совпадают → проверить построчно: термин в строке N EN файла должен иметь перевод в строке N RU файла (±2 строки допуск из-за различий в переносах)
- Расхождение = пропущенное предложение или лишнее вхождение → ошибка

**Формат отчёта по количеству:**
```
Термин: gamemaster (EN) → Мастер игры (RU)
  01_Introduction.md: EN=7, RU=7 ✓
  02_CharacterCreation.md: EN=14, RU=14 ✓
  03_System.md: EN=16, RU=15 ✗ (строка 42 EN не имеет соответствия в RU)
```

#### Агент 2: `verifier-structure`

**Фокус:** Структурная целостность

Для каждой пары EN/RU файлов проверяет:
- Количество файлов RU = EN (без глоссария)
- Структура папок RU = EN
- Количество заголовков каждого уровня (`#`, `##`, `###`, `####`): RU = EN
- Количество строк в таблицах: RU = EN
- Количество элементов в списках: RU = EN
- Количество blockquotes: RU = EN
- Количество пустых строк между разделами: ±1 допустимо

#### Агент 3: `verifier-quality`

**Фокус:** Качество перевода и оформление

Проверяет:
- Bold (`**text**`) — маркеры не потеряны и не добавлены лишние
- Italic (`*text*`) — маркеры сохранены
- Bold-italic (`**_text_**` или `***text***`) — сохранены
- Ссылки `[text](url)` — url не изменён
- Blockquotes (`>`) — структура сохранена
- Таблицы — правильное количество `|` в каждой строке
- Нет непереведённого EN текста (кроме: proper nouns, URL, code, числа)
- Нет двойных пробелов, сломанного форматирования
- Тире: длинное тире ` — `, а не ` -- ` или ` - `
- Числа без изменений (не переведены/не конвертированы)

### Шаг 3: Цикл верификации (минимум 3 раунда)

```
раунд = 1
ПОВТОРЯТЬ:
    1. Запустить всех 3 агентов ПАРАЛЛЕЛЬНО (через TaskCreate + Agent)
    2. Собрать результаты от всех агентов
    3. Объединить в общий отчёт:
       - Ошибки терминологии: [список]
       - Ошибки структуры: [список]
       - Ошибки качества: [список]
    4. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - Коммит: "Верификация контента {game} {version}: раунд {N}"
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл
    5. ЕСЛИ ошибок нет И раунд >= 3:
       - ВЫХОД из цикла
    6. ЕСЛИ ошибок нет И раунд < 3:
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл (нужно минимум 3 чистых проверки)
```

### Шаг 4: Отчёт

```
Верификация контента завершена:
- Файлов проверено: K
- Раундов: N
- Исправлений: M
- Статус: ✓ Все проверки пройдены

Детали по раундам:
- Раунд 1: X ошибок терминологии, Y структуры, Z качества → исправлено
- Раунд 2: ...
- Раунд 3: 0 ошибок ✓
```

## Технические требования

- Все 3 агента — **model: "opus"** (обязательно)
- Агенты запускаются **параллельно** через TeamCreate
- Минимум 3 раунда проверки даже если ошибок нет
- Большие файлы (>3000 строк) — чтение чанками
- Коммит только когда есть реальные исправления
- Цикл продолжается пока есть ошибки (без верхнего лимита)
- Сообщения коммитов на русском
