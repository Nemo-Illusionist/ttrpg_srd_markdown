# /verify-content — Верификация перевода контента

## Использование

```
/verify-content <game> <version>
```

Пример: `/verify-content daggerheart srd-1.0`

## Алгоритм

### Шаг 1: Подготовка

1. Найди EN контент: `src/{game}/{version}/en/**/*.md` (исключая `*_Glossary/`)
2. Найди RU контент: `src/{game}/{version}/ru/**/*.md` (исключая `*_Glossary/`)
3. Построй словарь терминов из RU глоссария (как в `/translate-content`)
4. Если RU контент не найден — предложи сначала `/translate-content`

### Шаг 2: Запуск верификатора

Создай **Agent** (обязательно **model: "opus"**) для проверки всех файлов.

Агент проверяет каждый RU файл по следующим критериям:

#### A. Терминологическая точность

- Извлеки все EN термины из словаря глоссария
- Для каждого термина: проверь что в RU файле используется ТОЛЬКО утверждённый RU перевод
- **Grep** EN терминов в RU файлах — не должно быть совпадений (кроме колонки "Оригинал" в глоссарии)
- Исключения: proper nouns, названия файлов, URL, code blocks

#### B. Структурная целостность

Для каждой пары EN/RU файлов:
- Количество заголовков каждого уровня (`#`, `##`, `###`, `####`): RU = EN
- Количество строк в таблицах: RU = EN
- Количество элементов в списках: RU = EN
- Количество blockquotes: RU = EN
- Количество пустых строк между разделами: ±1 допустимо

#### C. Форматирование

- Bold (`**text**`) — маркеры не потеряны и не добавлены лишние
- Italic (`*text*`) — маркеры сохранены
- Bold-italic (`**_text_**` или `***text***`) — сохранены
- Ссылки `[text](url)` — url не изменён
- Blockquotes (`>`) — структура сохранена
- Таблицы — правильное количество `|` в каждой строке

#### D. Полнота

- Количество файлов RU = EN (без глоссария)
- Структура папок RU = EN
- Нет непереведённого EN текста (кроме: proper nouns, URL, code, числа)

### Шаг 3: Цикл исправлений

```
раунд = 1
ПОВТОРЯТЬ:
    1. Агент (opus) проверяет ВСЕ RU файлы по критериям A-D
    2. Собрать список ошибок:
       - Терминология: [файл: строка, ожидалось X, найдено Y]
       - Структура: [файл: тип расхождения]
       - Форматирование: [файл: описание проблемы]
       - Полнота: [отсутствующие файлы/разделы]
    3. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - Коммит: "Верификация контента {game} {version}: раунд {N}"
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл
    4. ЕСЛИ ошибок нет:
       - ВЫХОД из цикла (минимум 1 раунд проверки)
```

### Шаг 4: Отчёт

```
Верификация контента завершена:
- Файлов проверено: K
- Раундов: N
- Исправлений: M
- Статус: ✓ Все проверки пройдены

Детали:
- Терминология: все термины по глоссарию ✓
- Структура: заголовки/таблицы/списки совпадают ✓
- Форматирование: bold/italic/links сохранены ✓
- Полнота: все файлы и разделы на месте ✓
```

## Технические требования

- Агент верификатор — **model: "opus"** (обязательно)
- Большие файлы (>3000 строк) — чтение чанками
- Коммит только когда есть реальные исправления
- Цикл продолжается пока есть ошибки (без верхнего лимита)
- Сообщения коммитов на русском
