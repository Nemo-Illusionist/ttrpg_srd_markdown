# /verify-content — Верификация перевода контента

## Использование

```
/verify-content <game> <version> [file]
```

- `/verify-content daggerheart srd-1.0` — весь перевод (3 агента, 3+ раундов)
- `/verify-content daggerheart srd-1.0 03_System.md` — одна страница (1 агент, fix-цикл)

## Правила проверки

Чеклисты:
- **`.claude/rules/translation-validation.md`** — структура, терминология, оформление, локализация
- **`.claude/rules/translation-quality-review.md`** — редакторская проверка качества (для `verifier-quality`)

Агенты ОБЯЗАНЫ прочитать свои rules файлы перед началом проверки.

## Алгоритм

### Шаг 1: Подготовка

1. Найди EN контент: `src/{game}/{version}/en/**/*.md` (исключая `*_Glossary/`)
2. Найди RU контент: `src/{game}/{version}/ru/**/*.md` (исключая `*_Glossary/`)
3. Построй словарь терминов из `src/{game}/translate/` и `src/translate/` (см. `.claude/rules/translate-dictionaries.md`)
4. Если RU контент не найден — предложи сначала `/translate-content`
5. Определи режим:
   - `[file]` указан → **режим одной страницы**
   - `[file]` не указан → **режим полной верификации**

---

### Режим полной верификации (без `[file]`)

#### Шаг 2: Создание команды верификаторов

Используй **TeamCreate** для создания команды `verify-content-{game}`.

Создай 3 агента через **Agent** (все с **model: "opus"**):

##### Агент 1: `verifier-terminology`

**Фокус:** Терминологическая точность, количественное соответствие и кросс-версионная консистентность

Проверяет разделы **2. Терминологическое соответствие** + **7. Кросс-версионная консистентность** из `.claude/rules/translation-validation.md`.

##### Агент 2: `verifier-structure`

**Фокус:** Структурная целостность

Проверяет раздел **1. Структурное соответствие** из `.claude/rules/translation-validation.md`.

##### Агент 3: `verifier-quality`

**Фокус:** Качество перевода, оформление и локализация

Проверяет разделы **3. Качество оформления** + **5. Качество локализации** из `.claude/rules/translation-validation.md`.

Дополнительно использует **`.claude/rules/translation-quality-review.md`** как системный промпт — жёсткая редакторская проверка: кальки, канцелярит, тяжеловесность, смысловые смещения, стиль rulebook.

#### Шаг 3: Цикл верификации (минимум 3 раунда)

```
раунд = 1
ПОВТОРЯТЬ:
    1. Запустить всех 3 агентов ПАРАЛЛЕЛЬНО (через TaskCreate + Agent)
    2. Собрать результаты от всех агентов
    3. Объединить в общий отчёт:
       - Ошибки терминологии: [список]
       - Ошибки структуры: [список]
       - Ошибки качества: [список]
    4. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - Если исправление меняет перевод термина → добавить запись в `src/{game}/translate/00_log.md`
       - Коммит: "Верификация контента {game} {version}: раунд {N}"
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл
    5. ЕСЛИ ошибок нет И раунд >= 3:
       - ВЫХОД из цикла
    6. ЕСЛИ ошибок нет И раунд < 3:
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл (нужно минимум 3 чистых проверки)
```

#### Шаг 4: Отчёт

```
Верификация контента завершена:
- Файлов проверено: K
- Раундов: N
- Исправлений: M
- Статус: ✓ Все проверки пройдены

Детали по раундам:
- Раунд 1: X ошибок терминологии, Y структуры, Z качества → исправлено
- Раунд 2: ...
- Раунд 3: 0 ошибок ✓
```

---

### Режим одной страницы (с `[file]`)

#### Шаг 2: Запуск верификатора

Запусти 1 opus-агент (`verifier-page`) через **Agent** с **model: "opus"**.

Агент получает:
- EN файл: `src/{game}/{version}/en/{file}`
- RU файл: `src/{game}/{version}/ru/{file}`
- Словарь терминов из глоссария
- Инструкция: прочитать `.claude/rules/translation-validation.md` и проверить ВСЕ пункты для этой пары файлов

#### Шаг 3: Fix-цикл

```
ПОВТОРЯТЬ:
    1. Агент проверяет пару файлов по ВСЕМУ чеклисту
    2. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - ПРОДОЛЖИТЬ цикл
    3. ЕСЛИ ошибок нет:
       - ВЫХОД из цикла
```

Коммит НЕ создаётся — вызывающий скилл сам решает когда коммитить.

#### Шаг 4: Отчёт

```
Верификация страницы {file}:
- Итераций: N
- Исправлений: M
- Статус: ✓ / ✗
```

## Терминологические решения

При обнаружении спорного перевода термина — проверь `src/{game}/translate/00_log.md` и `src/translate/00_log.md` (общие). Если решение задокументировано — оно имеет приоритет. Новые спорные решения — предложить пользователю и добавить в `src/{game}/translate/00_log.md`.

## Технические требования

- Все агенты — **model: "opus"** (обязательно)
- Полный режим: агенты запускаются **параллельно** через TeamCreate, минимум 3 раунда
- Режим одной страницы: 1 агент, цикл до 0 ошибок
- Большие файлы (>3000 строк) — чтение чанками
- Коммит только в полном режиме и только когда есть реальные исправления
- Цикл продолжается пока есть ошибки (без верхнего лимита)
- Сообщения коммитов на русском
