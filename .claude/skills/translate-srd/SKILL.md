# /translate-srd — Полный пайплайн перевода SRD

## Использование

```
/translate-srd <game> <version>
```

Пример: `/translate-srd daggerheart srd-1.0`

## Пайплайн

Выполняй фазы строго последовательно. Каждая фаза — отдельный skill.

### Phase 0: Проверка наличия EN файлов + создание ветки

1. Проверь существование `src/{game}/{version}/en/` и наличие `.md` файлов в ней
2. Если директория пуста или не существует:
   - Спроси пользователя через **AskUserQuestion**: "EN файлы для {game} {version} не найдены. Запустить импорт из PDF?"
   - Опции: "Да, запустить /import-srd" / "Нет, файлы появятся позже"
   - Если "Да" → вызови `/import-srd` и после его завершения продолжи
   - Если "Нет" → останови пайплайн
3. **Создай ветку перевода:**
   ```bash
   git checkout -b translate/{game}-{version}
   ```
   - Если ветка уже существует (продолжение прерванного пайплайна) → переключись на неё: `git checkout translate/{game}-{version}`
4. Все коммиты фаз 1-7 идут в эту ветку

```
✓ Phase 0 завершена: EN файлы найдены (K файлов), ветка translate/{game}-{version} создана
```

### Phase 0.5: Поиск существующих версий (кросс-версионная согласованность)

1. Найди все версии той же системы: `src/{game}/*/ru/*_Glossary/00_Glossary.md`
2. Исключи текущую версию `{version}`
3. Если найдены другие версии:
   - Прочитай их `00_Glossary.md` — это **reference-глоссарий**
   - Выведи пользователю:
   ```
   Найдены существующие переводы {game}:
   - {other_version}: N терминов в глоссарии
   Перевод будет согласован с существующими версиями.
   ```
   - Reference-глоссарий передаётся во все последующие фазы
4. Если других версий нет:
   ```
   Других версий {game} не найдено. Кросс-версионная согласованность не требуется.
   ```

```
✓ Phase 0.5 завершена: reference-глоссарий загружен (N терминов из M версий)
```

### Phase 1: Создание EN глоссария

```
→ /build-glossary {game} {version}
```

Вызови skill `build-glossary` с аргументами `{game} {version}`.

После завершения выведи:
```
✓ Phase 1 завершена: EN глоссарий создан
```

### Phase 2: Пофайловый перевод глоссария с верификацией

```
→ /translate-glossary {game} {version}
```

Вызови skill `translate-glossary` с аргументами `{game} {version}`.

Каждый файл глоссария: перевод → верификация opus-агентом → fix-цикл → коммит.

После завершения:
```
✓ Phase 2 завершена: RU глоссарий создан (пофайловая верификация пройдена)
```

### Phase 3: Полная верификация глоссария

```
→ /translate-verify {game} {version}
```

Вызови skill `translate-verify` с аргументами `{game} {version}`.

Это включает:
- 3 агента параллельно (структура, консистентность, качество), минимум 3 раунда
- Кросс-файловые проверки которые невозможны при пофайловой верификации
- Автоматические исправления с коммитами

После завершения:
```
✓ Phase 3 завершена: полная верификация глоссария (N раундов, M исправлений)
```

### Phase 4: Ручная проверка глоссария (цикл)

`/translate-verify` уже включает AskUserQuestion с циклом ручной проверки.
Этот этап завершается когда пользователь говорит "ОК".

```
✓ Phase 4 завершена: ручная проверка глоссария пройдена
```

### Phase 5: Пофайловый перевод контента с верификацией

```
→ /translate-content {game} {version}
```

Вызови skill `translate-content` с аргументами `{game} {version}`.

Каждый файл: перевод → ручная проверка структуры → верификация opus-агентом → fix-цикл → коммит.

После завершения:
```
✓ Phase 5 завершена: контент переведён (K файлов, пофайловая верификация пройдена)
```

### Phase 6: Полная верификация всего перевода

```
→ /verify-content {game} {version}
```

Вызови skill `verify-content` с аргументами `{game} {version}`.

Это включает:
- 3 агента параллельно (терминология, структура, качество)
- Минимум 3 раунда проверки
- Кросс-файловые проверки: консистентность терминов между ВСЕМИ файлами

После завершения:
```
✓ Phase 6 завершена: полная верификация контента (N раундов, M исправлений)
```

### Phase 7: Ручная проверка контента (цикл)

После полной верификации — пауза для ручной проверки.

1. Используй **AskUserQuestion**:
   - Вопрос: "Полная верификация контента завершена (N раундов, M исправлений). Проверьте файлы в `src/{game}/{version}/ru/` и скажите ОК или внесите правки."
   - Опции: "ОК, продолжаем" / "Внёс правки, перепроверить"

2. Если пользователь выбрал "Внёс правки":
   - Запустить `/verify-content {game} {version}` заново
   - Снова показать AskUserQuestion
   - Повторять пока не скажет "ОК"

3. Если пользователь выбрал "ОК" → продолжить

```
✓ Phase 7 завершена: ручная проверка контента пройдена
```

### Phase 7.5: Squash merge в main

1. Переключись на main:
   ```bash
   git checkout main
   ```
2. Squash merge ветки перевода:
   ```bash
   git merge --squash translate/{game}-{version}
   ```
3. Создай коммит:
   ```
   Перевод {game} {version}: глоссарий + контент (N файлов)
   ```
4. Удали ветку:
   ```bash
   git branch -d translate/{game}-{version}
   ```

```
✓ Phase 7.5 завершена: squash merge в main
```

### Phase 8: Интеграция в сайт

```
→ /integrate-srd {game} {version}
```

Вызови skill `integrate-srd` с аргументами `{game} {version}`.

Это включает:
- `src/site/index.md` — главная страница (EN + RU блоки для нового SRD)
- `.github/scripts/prepare_docs.sh` — команды копирования
- `.github/workflows/pages.yml` — CI деплой
- `src/site/mkdocs.yml` — навигация + nav_translations
- `.github/workflows/release-{game}.yml` — release workflow
- Релизный тег + push (с подтверждением пользователя)

После завершения:
```
✓ Phase 8 завершена: интеграция в сайт + релиз

Полный пайплайн завершён для {game} {version}:
- Phase 0.5: Кросс-версионная согласованность ✓
- Phase 1: EN глоссарий ✓
- Phase 2: RU глоссарий (пофайловая верификация) ✓
- Phase 3: Полная верификация глоссария (N раундов) ✓
- Phase 4: Ручная проверка глоссария ✓
- Phase 5: Перевод контента (K файлов, пофайловая верификация) ✓
- Phase 6: Полная верификация контента ✓
- Phase 7: Ручная проверка контента ✓
- Phase 8: Интеграция + релиз ✓
```

## Восстановление после сбоя

Если пайплайн прерван, можно перезапустить с любой фазы, вызвав соответствующий skill напрямую.

**Ветка:** при возобновлении проверь, существует ли ветка `translate/{game}-{version}`. Если да — переключись на неё перед продолжением. Если нет (фазы 1-7 не начинались или уже смержены) — создай новую.

- EN файлов нет → начни с `/import-srd` (Phase 0)
- EN глоссария нет → начни с `/build-glossary` (Phase 1)
- RU глоссария нет → начни с `/translate-glossary` (Phase 2)
- RU глоссарий есть, не верифицирован → начни с `/translate-verify` (Phase 3)
- Глоссарий проверен, контент не переведён → начни с `/translate-content` (Phase 5)
- Контент переведён → начни с `/verify-content` (Phase 6)
- Всё переведено и проверено → начни с `/integrate-srd` (Phase 8)

## Технические требования

- Все агенты во всех фазах — **model: "opus"**
- Фазы 1-7 работают в ветке `translate/{game}-{version}`, коммиты по файлам
- Phase 7.5: squash merge в main (один коммит в истории main)
- Phase 8 (интеграция) — уже в main
- Пайплайн останавливается на Phase 4 и Phase 7 для ручной проверки
- При ошибке в любой фазе — остановка и отчёт пользователю
- Сообщения коммитов на русском
