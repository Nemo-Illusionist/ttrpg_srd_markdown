# /translate-glossary — Перевод глоссария EN→RU

## Использование

```
/translate-glossary <game> <version>
```

Пример: `/translate-glossary daggerheart srd-1.0`

## Алгоритм

### Шаг 1: Поиск EN глоссария и reference-глоссариев

1. Найди директорию глоссария: `src/{game}/{version}/en/*_Glossary/` (Glob)
2. Если не найдена — предложи сначала запустить `/build-glossary`
3. Прочитай все файлы EN глоссария
4. **Поиск reference-глоссариев (кросс-версионная согласованность):**
   - Найди все `src/{game}/*/ru/*_Glossary/00_Glossary.md` (кроме текущей `{version}`)
   - Если найдены — прочитай и построй маппинг `{EN: RU}` из существующих переводов
   - Это **reference-словарь** — приоритетный источник перевода общих терминов
   - Правила: см. `.claude/rules/translation-validation.md`, раздел 7 (кросс-версионная консистентность)

### Шаг 2: Создание RU директории

1. Определи номер: тот же `{NN}` что у EN глоссария
2. Создай `src/{game}/{version}/ru/{NN}_Glossary/`

### Шаг 3: Перевод `00_Glossary.md` (ПЕРВЫМ — устанавливает терминологию)

1. Добавь раздел "Редакционные принципы" в начало файла:

```markdown
# Глоссарий

## Редакционные принципы

- Устоявшиеся термины из официальных переводов сохраняются
- Новые термины переводятся по смыслу, оригинал указывается в скобках при первом упоминании
- Имена собственные транслитерируются, оригинал в скобках
- Единообразие: один EN термин = один RU перевод во всех файлах
- Игромеханические термины с заглавной буквы (Сила, Ловкость, Испуганный)
```

2. **Каждая таблица получает колонку "Оригинал" вторым столбцом:**

```markdown
## Состояния

| Состояние | Оригинал | Описание |
|---|---|---|
| Ослеплённый | Blinded | Не может видеть... |
| Очарованный | Charmed | ... |
```

3. Сортировка строк по русскому алфавиту внутри каждой таблицы
4. Числа без изменений
5. Сохрани в `src/{game}/{version}/ru/{NN}_Glossary/00_Glossary.md`
6. **Верификация** — запусти opus-агент через **Agent** (model: "opus"):
   - Агент читает `.claude/rules/translation-validation.md` и проверяет пару EN/RU файлов по чеклисту
   - Для глоссария дополнительно: наличие колонки "Оригинал", сортировка по алфавиту
7. **Fix-цикл:** если агент нашёл ошибки → исправить → перезапустить верификацию → повторять до 0 ошибок
8. **Коммит** (только после чистой верификации): `Глоссарий RU {game} {version}: перевод 00_Glossary.md`

### Шаг 4: Перевод остальных файлов (01+)

Для каждого файла из EN глоссария (в порядке номеров):

1. Читай EN файл
2. Переводи, **строго следуя терминологии из 00_Glossary.md**
3. Формат таблиц:
   - **RU имя — первая колонка**
   - **EN оригинал — вторая колонка (всегда)**
   - Остальные колонки переведены

```markdown
| Заклинание | Оригинал | Уровень | Школа | Классы |
|---|---|---|---|---|
| Брызги кислоты | Acid Splash | Заговор | Воплощение | Чародей, Волшебник |
```

4. **Сортировка по русскому алфавиту** (по первой колонке)
5. Числа без изменений (уровни, КО, размеры)
6. Сохрани в `src/{game}/{version}/ru/{NN}_Glossary/{filename}`
7. **Верификация** — запусти opus-агент через **Agent** (model: "opus"):
   - Агент читает `.claude/rules/translation-validation.md` и проверяет пару EN/RU файлов по чеклисту
   - Для глоссария дополнительно: наличие колонки "Оригинал", совпадение EN-значений, сортировка по алфавиту
8. **Fix-цикл:** если агент нашёл ошибки → исправить → перезапустить верификацию → повторять до 0 ошибок
9. **Коммит** (только после чистой верификации): `Глоссарий RU {game} {version}: перевод {filename}`

### Шаг 5: Итоговый отчёт

```
Глоссарий RU создан:
- 00_Glossary.md: N категорий, M терминов
- 01_Spells.md: K заклинаний (EN: K) ✓
- 02_Monsters.md: L монстров (EN: L) ✓
...
Все количества совпадают с EN.
```

## Принципы перевода

- **Reference-словарь — приоритет.** Если термин уже переведён в другой версии той же системы → использовать тот же перевод
- **Один EN термин = один RU перевод** — консистентность во всём глоссарии и между версиями
- Устоявшиеся термины D&D/TTRPG → использовать общепринятые переводы
- Новые/уникальные термины → перевод по смыслу
- Имена собственные → транслитерация
- Игромеханические термины → с заглавной буквы
- Осознанное отклонение от reference-словаря допустимо, но требует обоснования и подтверждения пользователем

## Технические требования

- Все агенты используют **model: "opus"**
- Большие файлы (>3000 строк) — чтение чанками через offset/limit
- RU файл всегда содержит колонку "Оригинал" с EN именем
- Коммит после каждого файла
- Сообщения коммитов на русском
