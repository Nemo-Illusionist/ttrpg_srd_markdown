# /verify-import — Верификация полноты импорта

## Использование

```
/verify-import <game> <version>
```

Пример: `/verify-import brp srd-1.0`

## Предварительные условия

EN файлы должны существовать: `src/{game}/{version}/en/*.md`

Также нужен оригинальный PDF для сверки (спросить путь если неизвестен).

## Алгоритм

### Шаг 1: Подготовка

1. Найди все EN файлы: `src/{game}/{version}/en/**/*.md`
2. Если файлов нет — предложи сначала `/convert-pdf` + `/cleanup-artifacts`
3. Определи путь к оригинальному PDF (спросить если неизвестен)

### Шаг 2: Цикл верификации

Создай **Agent** (обязательно **model: "opus"**) для проверки.

```
раунд = 1
ПОВТОРЯТЬ:
    1. Агент проверяет ВСЕ EN файлы по критериям (см. ниже)
    2. Собрать список ошибок
    3. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - Коммит: "Импорт {game} {version}: верификация (раунд {N})"
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл
    4. ЕСЛИ ошибок нет:
       - ВЫХОД из цикла (минимум 1 раунд проверки)
```

### Критерии проверки

#### A. Полнота

- Все страницы PDF представлены в markdown
- Нет пропущенных глав или разделов
- Оглавление PDF (если есть) соответствует набору файлов

#### B. Структура

- Количество заголовков каждого уровня (`#`, `##`, `###`, `####`)
- Иерархия заголовков логична (нет `####` без `###` выше)
- Каждый файл начинается с `#` заголовка

#### C. Таблицы

- Все таблицы из PDF присутствуют в markdown
- Правильное количество строк и столбцов
- Заголовок таблицы + разделитель `|---|` + данные
- Нет сломанных строк таблиц (неравное кол-во `|`)

#### D. Списки

- Все списки на месте
- Количество элементов совпадает с PDF
- Нумерованные списки сохраняют нумерацию
- Вложенность списков корректна

#### E. Текст

- Нет пропущенных абзацев (выборочная сверка с PDF)
- Нет обрезанных предложений
- Нет мусора от конвертации (номера страниц, колонтитулы)

#### F. Форматирование

- Bold/italic/blockquotes консистентны
- Все `**` и `*` маркеры закрыты
- Blockquotes (`>`) не разорваны
- Code blocks (`` ` ``) закрыты
- Пустые строки между разделами

### Шаг 3: Пауза для ручной проверки

После прохождения всех автоматических проверок — пауза через **AskUserQuestion**:

```
Импорт завершён и верифицирован (N раундов, M исправлений).

Файлы: src/{game}/{version}/en/
- 00_Legal.md
- 01_....md
- ...

Проверьте файлы. Всё ОК или нужны правки?
```

Опции:
- "ОК, всё хорошо" → завершение
- "Нужны правки" → спросить какие правки, исправить, ещё раунд верификации

### Шаг 4: Итоговый отчёт

```
Верификация импорта завершена:
- Файлов проверено: K
- Раундов: N
- Исправлений: M
- Статус: ✓ Все проверки пройдены

Детали:
- Полнота: все страницы PDF покрыты ✓
- Структура: заголовки корректны ✓
- Таблицы: все на месте, структура верна ✓
- Списки: количество элементов совпадает ✓
- Текст: пропусков не обнаружено ✓
- Форматирование: маркеры закрыты, консистентно ✓
```

## Технические требования

- Агент верификатор — **model: "opus"** (обязательно)
- Большие файлы (>3000 строк) — чтение чанками через offset/limit
- PDF читается через Read tool (поддерживает PDF, параметр pages)
- Коммит только когда есть реальные исправления
- Цикл продолжается пока есть ошибки (без верхнего лимита)
- Сообщения коммитов на русском
