# /verify-import — Верификация полноты импорта

## Использование

```
/verify-import <game> <version>
```

Пример: `/verify-import brp srd-1.0`

## Правила проверки

Чеклист: **`.claude/rules/verify-import.md`** — классификация проблем, формат отчёта, hard failure rule.

Агент ОБЯЗАН прочитать этот файл перед началом проверки.

## Предварительные условия

- EN файлы: `src/{game}/{version}/en/**/*.md`
- Оригинальный PDF для сверки (спросить путь если неизвестен)

## Алгоритм

### Шаг 1: Автоматические проверки (скрипты)

Запусти скрипты ДО агента — они ловят механические проблемы быстрее:

```bash
# Структурная проверка markdown (таблицы, заголовки, bold)
python3 .claude/skills/verify-import/check_markdown.py src/{game}/{version}/en/
```

Если скрипт нашёл ошибки — исправь их до запуска агента.

### Шаг 2: Подготовка

1. Найди все EN файлы: `src/{game}/{version}/en/**/*.md`
2. Если файлов нет — предложи `/convert-pdf` + `/cleanup-artifacts`
3. Определи путь к оригинальному PDF

### Шаг 3: Цикл верификации (минимум 2 раунда)

```
раунд = 1
ПОВТОРЯТЬ:
    1. Агент проверяет ВСЕ EN файлы по критериям A-F (см. rules)
    2. Для проверки полноты (A): обязательно сверить с PDF:
       - Первые 5 страниц (оглавление, лицензия)
       - Середину документа (случайные 3 страницы)
       - Последние 5 страниц
    3. Собрать список ошибок
    4. ЕСЛИ есть ошибки:
       - Исправить все найденные проблемы
       - Коммит: "Импорт {game} {version}: верификация (раунд {N})"
       - раунд += 1
       - ПРОДОЛЖИТЬ цикл
    5. ЕСЛИ ошибок нет И раунд >= 2:
       - ВЫХОД из цикла ✓
    6. ЕСЛИ ошибок нет И раунд < 2:
       - раунд += 1
       - ПРОДОЛЖИТЬ (нужно минимум 2 чистых раунда)
```

### Критерии проверки (A-F)

Детальный чеклист: **`.claude/rules/verify-import.md`**

- **A. Полнота** — все страницы PDF покрыты, нет пропусков (обязательна сверка с PDF)
- **B. Структура** — иерархия заголовков, файлы начинаются с `#`
- **C. Таблицы** — все на месте, строки/столбцы совпадают с PDF
- **D. Списки** — количество элементов совпадает
- **E. Текст** — нет мусора, OCR-артефактов
- **F. Форматирование** — bold/italic закрыты, blockquotes целы

### Шаг 4: Пауза для ручной проверки

```
Импорт завершён и верифицирован (N раундов, M исправлений).

Файлы: src/{game}/{version}/en/
- 00_Legal.md
- 01_....md
...

Проверьте файлы. Всё ОК или нужны правки?
```

### Шаг 5: Итоговый отчёт

```
Верификация импорта завершена:
- Файлов проверено: K
- Раундов: N
- Исправлений: M
- Статус: ✓ / ✗
```

## Скрипты

| Скрипт | Что делает |
|--------|------------|
| `check_markdown.py` | Автоматические проверки: таблицы, заголовки, bold/italic, пустые файлы |

## Технические требования

- Агент — **model: "opus"**
- Большие файлы (>3000 строк) — чтение чанками
- PDF читается через Read tool (параметр `pages`)
- Коммит только при реальных исправлениях
- Минимум 2 раунда (1 проверка + 1 контрольный)
- Сообщения коммитов на русском
