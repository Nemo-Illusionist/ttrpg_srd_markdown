# Словари перевода (translate/)

Авторитетный источник терминологии для перевода. Словари первичны, `*_Glossary/` в версиях — публичная производная.

## Структура

### Уровень системы: `src/{game}/translate/`

```
logs/                        — лог терминологических решений
  {YYYY-MM-DD}_log.md        — записи за дату
01_dictionary_base.md        — базовые термины (характеристики, механики, условия)
02_dictionary_{entity}.md    — сущности (заклинания, монстры, оружие, ...)
03_dictionary_{entity}.md
...
```

### Общий уровень: `src/translate/`

```
logs/                        — общие решения (все системы)
  {YYYY-MM-DD}_log.md        — записи за дату
01_dictionary_base.md        — термины, общие для всех систем (минимальный набор)
```

## Нумерация

- `01` — базовый словарь
- `02+` — словари сущностей (по значимости/размеру)
- `logs/` — хронологические логи решений

## Категории терминов в базовом словаре (01)

Базовый словарь содержит **все повторяющиеся термины**, а не только игромеханические. Категории:

### Обязательные (ядро)
- Характеристики (Strength → Сила)
- Состояния (Blinded → Ослеплённый)
- Типы урона (Fire → Огонь)
- Классы (Fighter → Воин)
- Виды/расы (Elf → Эльф)
- Навыки (Stealth → Скрытность)
- Типы существ (Undead → Нежить)
- Ключевые механики (Saving Throw → Спасбросок, Hit Points → Хиты)

### Расширенные (для консистентности между агентами)
- **Названия разделов/глав** (Playing the Game → Правила игры) — для кросс-файловых ссылок
- **Названия подклассов** (Path of the Berserker → Путь берсерка, Circle of the Land → Круг Земли)
- **Названия ключевых умений классов** (Rage → Ярость, Sneak Attack → Скрытая атака, Wild Shape → Дикий облик)
- **Рекуррентные фразы** (Using a Higher-Level Spell Slot → Использование ячейки более высокого уровня, Cantrip Upgrade → Улучшение заговора)
- **Единицы и сокращения** (ft. → футов, GP → зм, SP → см, CP → мм)

### Принцип: если термин встречается в 2+ файлах — он должен быть в словаре.

Без словарной фиксации параллельные агенты перевода могут выбрать разные варианты (например, "Игровой процесс" vs "Правила игры" для Playing the Game).

## Формат таблиц

### Стандартный (одна версия: BRP, Daggerheart)

```markdown
| Оригинал | Перевод | Источник | Комментарий |
|---|---|---|---|
| Strength | Сила (СИЛ) | srd-1.0 | STR, 3D6 |
```

### Мультиверсионный (D&D: несколько версий)

```markdown
| Оригинал 5.2 | Оригинал 5.1 | Перевод | Источник 5.2 | Источник 5.1 | Комментарий |
|---|---|---|---|---|---|
| Strength | Strength | Сила (Сил.) | srd-5.2 | srd-5.1 | — |
| Sorcerer | — | Чародей | — | srd-5.1 | — |
```

Колонки `Оригинал {version}` — по одной на каждую версию. `—` если термин отсутствует в версии.
Колонки `Источник {version}` — указывают, в каком SRD-документе термин появился.

### Общий словарь (`src/translate/01_dictionary_base.md`)

```markdown
| Оригинал | Перевод | Комментарий |
|---|---|---|
| Campaign | Кампания | |
```

Без колонки `Источник` — термины применимы ко всем системам.

## Сортировка

Строки в таблицах — по **русскому алфавиту** (колонка `Перевод`).

## Лог решений (`logs/`)

Логи хранятся в `src/{game}/translate/logs/{YYYY-MM-DD}_log.md` (и `src/translate/logs/` для общих).

Каждый файл — записи за одну дату. Если файл за дату уже существует — дописывать в конец.

### Когда добавлять запись

- Спорный выбор перевода (несколько равноценных вариантов)
- Отклонение от устоявшегося перевода или reference другой версии
- Агент качества (верификатор) меняет перевод термина — любое изменение терминологии по результатам верификации
- Пользователь явно утверждает нестандартное решение
- Любое изменение уже принятого перевода (revision)

### Типы записей

- **decision** — первичное решение (выбор перевода при создании словаря)
- **revision** — изменение ранее принятого перевода (обязательно обновить dictionary)
- **exception** — осознанное отклонение для конкретного контекста
- **deprecated** — отказ от термина (с указанием замены)

### Формат записи

```markdown
### EN термин → RU перевод

- **Тип:** decision | revision | exception | deprecated
- **Решение:** выбранный вариант
- **Причина:** обоснование
- **Отвергнуто:** альтернативные варианты (если были)
- **Затронуто:** файлы/этапы (для revision/deprecated)
```

### Формат файла лога

```markdown
# Лог терминологических решений — {Game} — {YYYY-MM-DD}

### Term1 → Перевод1
- **Тип:** decision
- **Решение:** ...
- **Причина:** ...

### Term2 → Перевод2
- **Тип:** revision
- **Решение:** ...
- **Причина:** ...
- **Затронуто:** 5 файлов в srd-1.0/ru/
```

## Связь с `*_Glossary/`

- **`translate/`** — источник истины для терминологии (внутренний, для pipeline)
- **`*_Glossary/`** — публичная часть перевода SRD (для сайта)
- При валидации: термины в `*_Glossary/` проверяются на соответствие `translate/`
- При конфликте: `translate/` побеждает

## Приоритет источников и conflict resolution

**Правило:** более поздняя явная запись в логе побеждает любой словарь. Системный словарь побеждает общий, если нет записи в логе.

Приоритет (от высшего к низшему):

1. `src/{game}/translate/logs/` — последняя запись по дате (абсолютный приоритет)
2. `src/translate/logs/` — общие решения
3. `src/{game}/translate/01_dictionary_base.md` — системный словарь
4. `src/translate/01_dictionary_base.md` — общий словарь
5. `src/{game}/translate/02+_dictionary_*.md` — словари сущностей
6. `src/{game}/{version}/ru/*_Glossary/` — глоссарий версии (производный)

**Инвариант:** dictionary и logs не должны расходиться. Каждый `revision` в логе обязан иметь соответствующее обновление в dictionary. Подробнее: `.claude/rules/terminology-propagation.md`.

## Жизненный цикл

1. `/build-glossary` — создаёт EN глоссарий из исходников
2. `/translate-glossary` — переводит глоссарий + создаёт/обновляет словари в `translate/`
3. `/translate-content` — использует `translate/` как источник терминологии
4. `/verify-content` — валидирует перевод против `translate/`
