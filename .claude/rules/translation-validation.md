# Правила валидации перевода

Единый чеклист для проверки перевода SRD. Используется в `/verify-content`, `/translate-content`, `/translate-verify` и `/translate-glossary`.

## 1. Структурное соответствие

Для каждой пары EN/RU файлов:

- Количество строк: RU = EN (допуск ±1 на пустые строки)
- Количество заголовков каждого уровня (`#`, `##`, `###`, `####`): RU = EN
- Количество строк в таблицах: RU = EN
- Количество столбцов в таблицах (кол-во `|` в строке): RU = EN
- Количество элементов в списках: RU = EN
- Количество blockquotes (`>`): RU = EN
- Структура папок и файлов RU = EN (без глоссария `*_Glossary/`)

## 2. Терминологическое соответствие

Источник терминов: `src/{game}/translate/` (авторитетный). См. `.claude/rules/translate-dictionaries.md`.

- Все термины из словарей использованы правильно (EN→RU по словарю)
- Нет EN терминов в RU файле (исключения: имена собственные, URL, code blocks, числа)
- Один термин = один перевод во всех файлах (нет синонимов)
- Количество вхождений ключевых терминов: EN ≈ RU для каждой пары файлов (с допуском на русские падежные формы)
- При расхождении количества — построчная проверка: термин в строке N EN → перевод в строке N RU (±2 строки допуск)

### Формат отчёта по количеству

```
Термин: gamemaster (EN) → Мастер игры (RU)
  01_Introduction.md: EN=7, RU=7 ✓
  02_CharacterCreation.md: EN=14, RU=14 ✓
  03_System.md: EN=16, RU=15 ✗ (строка 42 EN не имеет соответствия в RU)
```

## 3. Качество оформления

- Bold (`**text**`) — маркеры не потеряны и не добавлены лишние
- Italic (`*text*`) — маркеры сохранены
- Bold-italic (`**_text_**` или `***text***`) — сохранены
- Ссылки `[text](url)` — url не изменён
- Blockquotes (`>`) — структура сохранена
- Тире: длинное тире ` — `, а не ` -- ` или ` - `
- Нет двойных пробелов
- Нет сломанного форматирования
- Числа без изменений (не переведены/не конвертированы)
- Нет непереведённого EN текста (кроме: имена собственные, URL, code blocks, числа)

## 5. Качество локализации

- Нет буквальных калек с английского (конструкции типа "является X" вместо "это X", "в случае если" вместо "если")
- Нет канцелярита (нагромождение отглагольных существительных, цепочки родительных падежей)
- Нет стилистической тяжеловесности — фразы сжатые и ясные
- Правиловой текст звучит как rulebook, а не как художественный пересказ
- Художественные описания — повествовательный стиль, не сухой технический
- Нет потери или добавления смысловых оттенков при переводе
- Аббревиатуры единообразны во всех файлах
- Названия характеристик, навыков, механик — строго по глоссарию
- Броски, проверки и формулы — технически точный перевод
- Перевод звучит как профессиональная локализация, а не как сырой перевод

Подробный чеклист редакторской проверки: **`.claude/rules/translation-quality-review.md`**.

## 6. Специфика глоссария

Дополнительные правила для файлов в `*_Glossary/`. Применяются поверх разделов 1-5.

### Структура
- Количество файлов RU = EN
- Количество строк в каждой таблице RU = EN
- Количество столбцов: RU = EN + 1 (колонка "Оригинал" — вторая)
- Наличие колонки "Оригинал" во ВСЕХ таблицах

### Точность
- EN-значения в колонке "Оригинал" точно совпадают с EN-файлами
- Заголовки разделов соответствуют EN
- Нет пропущенных сущностей

### Консистентность (кросс-файловая)
- Один EN термин → один RU термин во ВСЕХ файлах глоссария
- `00_Glossary.md` — авторитетный источник, 01+ файлы подчиняются ему
- Нет синонимов (один термин переведён двумя способами)
- Нет омонимов (два разных EN термина переведены одинаково)
- Нет дубликатов: каждый EN термин встречается в глоссарии ровно один раз (нет повторных строк в таблицах)

### Оформление
- Редакционные принципы в `00_Glossary.md` присутствуют
- Капитализация: игромеханические термины с заглавной
- Сортировка по русскому алфавиту в каждой таблице
- Нет непереведённого текста (кроме колонки "Оригинал")

## 7. Кросс-версионная консистентность

При наличии нескольких версий одной системы (например, `dnd/srd-5.1` и `dnd/srd-5.2`):

### Обнаружение

Кросс-версионная консистентность обеспечивается словарями `src/{game}/translate/` — они общие для всех версий системы.

1. Прочитай `src/{game}/translate/01_dictionary_base.md` + `02+_dictionary_*.md`
2. Прочитай логи `src/{game}/translate/logs/`
3. Сверь термины в текущей версии с dictionary

### Правила

- Все версии одной системы используют **один dictionary** → термины единообразны по определению
- Если в RU файлах версии найден перевод, отличающийся от dictionary → **error**
- Осознанное изменение перевода допустимо, но требует:
  - Процедуры из `.claude/rules/terminology-propagation.md`
  - Обновления перевода во ВСЕХ версиях системы
  - Подтверждения пользователем

### Что проверять

- Совпадение перевода общих терминов между версиями
- Совпадение аббревиатур
- Совпадение имён собственных (NPC, локации, божества)
- Совпадение перевода названий заклинаний, монстров, предметов, которые есть в обеих версиях

### Формат отчёта

```
Кросс-версионная проверка: dnd/srd-5.2 vs dnd/srd-5.1

Общих терминов: 142
Совпадений: 140 ✓
Расхождений: 2 ✗
  - "Proficiency Bonus": srd-5.1="Бонус мастерства", srd-5.2="Бонус владения" ← КОНФЛИКТ
  - "Hit Points": srd-5.1="Хиты", srd-5.2="Очки здоровья" ← КОНФЛИКТ
```

## 8. Классификация ошибок по severity

При отчёте каждая ошибка получает severity:

- **Критическая ошибка перевода** — смысл искажён, потерян или противоположен оригиналу
- **Смысловое смещение** — смысл частично сдвинут, правило работает иначе
- **Терминологическое несоответствие** — неверный термин или непоследовательность
- **Кросс-версионный конфликт** — термин переведён иначе, чем в другой версии той же системы
- **Пропущенный контент** — предложение, абзац или элемент списка не переведён
- **Стилистическое отклонение** — калька, канцелярит, тяжеловесность
- **Ошибка оформления** — форматирование, пробелы, тире
- **Мелкая шероховатость** — незначительная проблема формулировки

**Hard failure:** при наличии хотя бы одной критической ошибки, пропущенного контента или неразрешённого кросс-версионного конфликта — верификация не пройдена
