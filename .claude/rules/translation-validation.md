# Правила валидации перевода

Единый чеклист для проверки перевода SRD. Используется в `/verify-content` и `/translate-content`.

## 1. Структурное соответствие

Для каждой пары EN/RU файлов:

- Количество строк: RU = EN (допуск ±1 на пустые строки)
- Количество заголовков каждого уровня (`#`, `##`, `###`, `####`): RU = EN
- Количество строк в таблицах: RU = EN
- Количество столбцов в таблицах (кол-во `|` в строке): RU = EN
- Количество элементов в списках: RU = EN
- Количество blockquotes (`>`): RU = EN
- Структура папок и файлов RU = EN (без глоссария `*_Glossary/`)

## 2. Терминологическое соответствие

Источник терминов: `*_Glossary/00_Glossary.md` (авторитетный).

- Все термины из глоссария использованы правильно (EN→RU по словарю)
- Нет EN терминов в RU файле (исключения: имена собственные, URL, code blocks, числа)
- Один термин = один перевод во всех файлах (нет синонимов)
- Количество вхождений ключевых терминов: EN ≈ RU для каждой пары файлов (с допуском на русские падежные формы)
- При расхождении количества — построчная проверка: термин в строке N EN → перевод в строке N RU (±2 строки допуск)

### Формат отчёта по количеству

```
Термин: gamemaster (EN) → Мастер игры (RU)
  01_Introduction.md: EN=7, RU=7 ✓
  02_CharacterCreation.md: EN=14, RU=14 ✓
  03_System.md: EN=16, RU=15 ✗ (строка 42 EN не имеет соответствия в RU)
```

## 3. Качество оформления

- Bold (`**text**`) — маркеры не потеряны и не добавлены лишние
- Italic (`*text*`) — маркеры сохранены
- Bold-italic (`**_text_**` или `***text***`) — сохранены
- Ссылки `[text](url)` — url не изменён
- Blockquotes (`>`) — структура сохранена
- Тире: длинное тире ` — `, а не ` -- ` или ` - `
- Нет двойных пробелов
- Нет сломанного форматирования
- Числа без изменений (не переведены/не конвертированы)
- Нет непереведённого EN текста (кроме: имена собственные, URL, code blocks, числа)
