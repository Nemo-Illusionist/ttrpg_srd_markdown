# Правила чистки артефактов PDF→Markdown

Единый чеклист для Фазы C скилла `/cleanup-artifacts`. Агент ОБЯЗАН прочитать этот файл перед началом чистки.

Цель: нормализовать markdown так, чтобы он выглядел как документ, изначально написанный в markdown, а не извлечённый из PDF.

**НЕ переписывать контент. Только исправлять структурные и форматные дефекты.**

## 1. Разрывы строк внутри предложений

PDF часто вставляет переносы строк посередине предложения.

Правило: если строка обрывается и следующая начинается с маленькой буквы — склеить.

```
ПЛОХО:
To play, a group needs at least one set of these dice, though it is easier
and more convenient if each player has a
set.

ХОРОШО:
To play, a group needs at least one set of these dice, though it is easier and more convenient if each player has a set.
```

## 2. Дефисные переносы слов

Правило: если слово заканчивается на `-` и следующая строка продолжает слово — склеить.

```
ПЛОХО:
royalty-
free

ХОРОШО:
royalty-free
```

## 3. Разбитые составные слова

Правило: если слово разбито на две строки без дефиса и это явно искусственный разрыв — склеить.

```
ПЛОХО:          ХОРОШО:
four            four-sided
sided

role            roleplaying
playing

six             six-sided
sided
```

## 4. Номера страниц и колонтитулы

Удалять:
- Числа на отдельных строках между разделами (номера страниц)
- Повторяющийся текст на каждой «странице» (колонтитулы, headers/footers)
- Watermarks

## 5. OCR-маркеры и метаданные конвертеров

Удалять маркеры типа:
- `<PARSED TEXT FOR PAGE: X / Y>`
- `<!-- page X -->`
- Любые теги метаданных, добавленные конвертерами

## 6. Image placeholders

Удалять декоративные изображения:
- `![](_page_2_Picture_16.jpeg)` — логотипы, орнаменты, разделители
- Сохранять только осмысленные диаграммы и схемы (если таковые есть)

## 7. Таблицы

- Отсутствующие `|` на краях строк — добавить
- Неравное количество столбцов в строках — выровнять
- Разделитель `|---|` отсутствует после заголовка — добавить
- Разорванные строки таблиц — склеить

## 8. Сломанные списки

Нормализовать смешанные стили нумерации:

```
ПЛОХО:
- 1. Definitions:
- (a) "Contributors" means

ХОРОШО:
1. Definitions:
   (a) "Contributors" means
```

Bullet lists, нумерованные списки — привести к единому стилю markdown.

## 9. Заголовки

- Заголовки без `#` — добавить правильный уровень
- Разные стили (`#### **1. INTRODUCTION**`, `### 1. INTRODUCTION`, `1. INTRODUCTION`) — нормализовать
- Сохранять иерархию уровней
- Убирать лишнее форматирование из заголовков (bold, uppercase)

## 10. Форматирование

- Разорванные bold/italic (`**text` без закрывающего `**`) — исправить
- Артефакты OCR: лигатуры `ﬁ`→`fi`, `ﬂ`→`fl`, `ﬀ`→`ff`
- Soft hyphens (`\u00AD`) — удалить
- Двойные/лишние пробелы — нормализовать
- Trailing whitespace — удалить
- Потерянные пустые строки между разделами — восстановить
- Избыточные пустые строки (3+ подряд) — сократить до 2

## 11. Нормализация (опционально)

- Множественные пробелы → один
- Тире: `--` → `—` (em dash), ` - ` в середине предложения → ` — `
- Кавычки: прямые → типографские (если в документе используются типографские)

## Не трогать

- Dice notation: `3D6`, `D10`, `D100`, `1D8+1` и т.д.
- Rule references и кросс-ссылки
- System terms и игровые термины
- Числа и формулы
- Цитаты и выделенный текст
- Смысл и содержание текста
