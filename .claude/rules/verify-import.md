# Правила верификации импорта (Import Verification)

Используется в `/verify-import`. Агент выполняет аудит импорта, а не редактирование.

Цель: обнаружить дефекты конвертации между PDF и Markdown.

## Что проверять

### A. Полнота

- Все страницы PDF представлены в markdown
- Нет пропущенных глав или разделов
- Оглавление PDF (если есть) соответствует набору файлов
- Нет пропущенных абзацев (выборочная сверка с PDF)
- Нет обрезанных предложений
- Нет дублированного текста

### B. Структура

- Количество заголовков каждого уровня (`#`, `##`, `###`, `####`)
- Иерархия заголовков логична (нет `####` без `###` выше)
- Каждый файл начинается с `#` заголовка
- Правильный порядок абзацев и секций
- Неправильная иерархия секций

### C. Таблицы

- Все таблицы из PDF присутствуют в markdown
- Правильное количество строк и столбцов
- Заголовок таблицы + разделитель `|---|` + данные
- Нет сломанных строк таблиц (неравное кол-во `|`)

### D. Списки

- Все списки на месте
- Количество элементов совпадает с PDF
- Нумерованные списки сохраняют нумерацию
- Вложенность списков корректна

### E. Текст

- Нет мусора от конвертации (номера страниц, колонтитулы)
- Нет OCR-артефактов (битые символы)
- Нет искажения смысла из-за склейки строк

### F. Форматирование

- Bold/italic/blockquotes консистентны
- Все `**` и `*` маркеры закрыты
- Blockquotes (`>`) не разорваны
- Code blocks (`` ` ``) закрыты
- Пустые строки между разделами

## Классификация проблем

- **Критический дефект импорта** — пропущена секция, таблица, или смысл искажён
- **Пропущенный контент** — пропущен абзац или предложение
- **Неправильный порядок блоков** — перемешанные абзацы/секции
- **Структурное несоответствие** — потерянный заголовок, сломанный список
- **Ошибка форматирования** — битая таблица, разорванный bold/italic
- **Безобидная нормализация** — незначительные различия в пробелах/пустых строках

## Формат отчёта

Для каждой проблемы:
- Доказательство из PDF (страница, позиция)
- Доказательство из Markdown (файл, строка)
- Severity (из классификации выше)
- Предложенное исправление

## Hard failure rule

Если обнаружен хотя бы один пропущенный абзац или секция — верификация ПАДАЕТ.

Pipeline НЕ продолжается к глоссарию и переводу пока полнота импорта не подтверждена.
