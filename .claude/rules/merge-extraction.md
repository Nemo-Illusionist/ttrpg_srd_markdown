# Правила сведения результатов PDF-конвертеров

Единый чеклист для Фазы A скилла `/cleanup-artifacts`. Агент ОБЯЗАН прочитать этот файл перед началом сведения.

Цель: реконструировать максимально точную версию оригинального документа из нескольких markdown-файлов, сгенерированных разными конвертерами из одного PDF.

## Принципы выбора

Не сливать контент слепо. Использовать семантическое сравнение.

1. **Длинный связный абзац лучше разбитого** — если в одной версии абзац цельный, а в другой разбит переносами строк, брать цельный
2. **Чистый текст лучше текста с OCR-артефактами** — если одна версия содержит лигатуры, мусор, битые символы, брать чистую
3. **Правильные заголовки приоритетнее** — версия с корректными уровнями `#` и иерархией лучше
4. **Правильные списки приоритетнее** — версия с сохранённой нумерацией и вложенностью лучше
5. **Правильные таблицы приоритетнее** — версия с корректной структурой `|` и выравниванием столбцов лучше
6. **Порядок разделов = оригинал** — сохранять порядок разделов как в PDF
7. **Дубликаты от перекрытия страниц — удалять** — конвертеры могут дублировать текст на стыках страниц

## Эвристики по конвертерам

Типичные сильные/слабые стороны (уточняются для каждого PDF):

| Конвертер | Обычно хорош для | Обычно слаб в |
|---|---|---|
| marker | Заголовки (правильные уровни `#`), общая структура | Сложные таблицы |
| pymupdf4llm | Текст, абзацы | Форматирование, таблицы |
| docling | Таблицы (структура) | Заголовки, иногда порядок секций |

**Это эвристики, не правила.** Для каждого конкретного PDF сравнивай все три и выбирай лучшее.

## Порядок работы

1. Прочитать все три файла (чанками если большие)
2. Определить структуру: главы и разделы по заголовкам
3. Сверить с оглавлением PDF (если есть)
4. Для каждого раздела: сравнить три версии, выбрать лучшую
5. При расхождениях — сверять с оригинальным PDF
6. Собрать единый markdown

## При расхождениях

Если все три версии отличаются и непонятно какая верна:
- Сверить с PDF (через Read tool)
- Если PDF нечитаем в этом месте — взять наиболее связную версию
- Отметить спорное место комментарием `<!-- REVIEW: расхождение конвертеров -->`
